================================================================================
                    ■ AGENT 1: SPEC AGENT (스펙 분석가) ■
================================================================================

【역할】
제품 스펙을 분석하여 "혜택 언어"로 번역

【입력 데이터】
- productinfo.txt (제품 상세 정보)

【출력 형식】
┌─────────────────────────────────────────────────────────────────────────────┐
│  스펙              │  혜택 번역                    │  후킹 점수 (1-10)      │
├─────────────────────────────────────────────────────────────────────────────┤
│  RTX 5060          │  "최신작 게임 풀옵 가능"      │  8                      │
│  144Hz 디스플레이  │  "끊김 없는 부드러운 게임"    │  7                      │
│  AI 칩 LA1         │  "AI로 더 똑똑하게"          │  6                      │
│  PD 충전 100W      │  "어댑터 없이 충전"          │  5                      │
│  Hyperchamber 쿨링 │  "뜨거워지지 않는 노트북"    │  9                      │
└─────────────────────────────────────────────────────────────────────────────┘

【핵심 지침】
1. 숫자 → 일상 언어로 변환
   - "16GB RAM" → "크롬 탭 50개 띄워도 버벅임 없음"
   - "512GB SSD" → "게임 5-6개는 거뜬히"
   
2. 비교 포인트 도출
   - 경쟁 제품 대비 강점: "LG 그램보다 성능, 맥북보다 가격"
   
3. 차별화 스펙 우선순위화
   - 1위: 쿨링 성능 (리뷰에서 호평 많음)
   - 2위: 가격 (125만원 특가)
   - 3위: 디스플레이 (144Hz, 100% sRGB)


================================================================================
                    ■ AGENT 2: REVIEW AGENT (리뷰 분석가) ■
================================================================================

【역할】
실제 구매자 리뷰에서 신뢰도 높은 키워드와 스토리 추출

【입력 데이터】
- review.txt (쿠팡 리뷰 157개 분석)

【출력 형식】
┌─────────────────────────────────────────────────────────────────────────────┐
│  카테고리      │  핵심 인용                              │  빈도     │
├─────────────────────────────────────────────────────────────────────────────┤
│  가성비        │  "125만원에 이 성능 말이 안 됨"         │  ⭐⭐⭐⭐⭐   │
│  쿨링          │  "CPU GPU 온도 80도 안 넘음"           │  ⭐⭐⭐⭐⭐   │
│  게임성능      │  "끊김 없이 부드럽게 돌아감"           │  ⭐⭐⭐⭐    │
│  무게          │  호불호 갈림 (적당해요 68%)            │  ⭐⭐⭐     │
│  단점          │  "생각보다 무거움", "포장 상태"        │  ⚠️        │
└─────────────────────────────────────────────────────────────────────────────┘

【핵심 지침】
1. 실제 사용자 언어 그대로 활용
   - "두마리 토끼 다 잡은 노트북"
   - "친구꺼 보고 발열 때문에 이거 바로 샀어요"
   
2. 부정 리뷰 분석 → 반전 소재로 활용
   - "무겁다" → "무거운 만큼 쿨링 뺴곡히 들어감"
   
3. 추천 대상 도출
   - 게이밍 83%, 그래픽 작업 10%, 학습용 5%
   - 타겟: 가성비 게이밍 노트북 찾는 20-30대

4. 신뢰도 높은 리뷰 선별
   - "도움됨" 1개 이상
   - 사진 첨부된 리뷰


================================================================================
                    ■ AGENT 3: STYLE AGENT (스타일 코치) ■
================================================================================

【역할】
나야리뷰 스타일 분석 + 차별화 요소 제안

【입력 데이터】
- 쇼츠대본_분석.txt (대본 분석)
- 쇼츠대본_나야리뷰.txt (참고 대본)

【출력 형식】

■ 나야리뷰 스타일 핵심 패턴
┌─────────────────────────────────────────────────────────────────────────────┐
│  요소           │  나야리뷰 패턴                │  우리만의 변형            │
├─────────────────────────────────────────────────────────────────────────────┤
│  후킹           │  "[제품]이 왜 좋은지 딱"      │  "[제품] 진짜 살만해?"    │
│  스펙전달       │  장점 → 스펙 순서             │  리뷰 인용 → 스펙 검증    │
│  비유           │  "포르쉐를 경차로"            │  일상 밈/트렌드 활용      │
│  CTA            │  "왼쪽 상품 보기"             │  "최저가 링크 걸어둠"     │
│  마무리         │  감성적 한마디                │  솔직한 단점+추천 조건    │
└─────────────────────────────────────────────────────────────────────────────┘

■ 우리만의 차별화 포인트 (🔥 킥)
┌─────────────────────────────────────────────────────────────────────────────┐
│  🔥 킥 1: 가격 추적기 데이터 활용                                          │
│  → "쿠팡 한 달 추적해봤는데 지금이 진짜 최저야"                            │
│  → "600명이 이미 샀어, 물량 얼마 없어"                                     │
│  → 가격 변동 그래프 화면 삽입                                              │
├─────────────────────────────────────────────────────────────────────────────┤
│  🔥 킥 2: 솔직한 단점 언급                                                 │
│  → "무게 2.4kg 솔직히 무거워, 근데 그만큼 쿨링이 미쳤어"                   │
│  → 신뢰도 상승, 과대광고 느낌 제거                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│  🔥 킥 3: 실제 리뷰 인용                                                   │
│  → "157개 리뷰 중 91%가 5점, 근데 1점 리뷰도 봤어"                        │
│  → 리뷰 화면 캡처 삽입                                                    │
├─────────────────────────────────────────────────────────────────────────────┤
│  🔥 킥 4: 구매 조건 명시                                                   │
│  → "이 가격 아니면 안 삼, 정가면 다른 거 사"                              │
│  → 가격대별 추천 분기                                                     │
└─────────────────────────────────────────────────────────────────────────────┘


================================================================================
                    ■ BOSS AGENT: 최종 대본 작성 ■
================================================================================

【역할】
각 에이전트 출력을 종합하여 최종 대본 생성

【대본 구조 템플릿】

```
[0-3초] 🎯 후킹
────────────────────────────────────────
{STYLE_AGENT.hook} + {REVIEW_AGENT.top_keyword}
예) "가성비 게이밍 노트북? 리뷰 157개 직접 분석해봤어"

[3-10초] 📊 핵심 스펙 (혜택 언어)
────────────────────────────────────────
{SPEC_AGENT.top3_benefits}
예) "RTX 5060에 144Hz 화면, 게임 풀옵 끊김 없이 돌아감"

[10-20초] 💬 리뷰 검증
────────────────────────────────────────
{REVIEW_AGENT.top_quote}
예) "실제로 산 사람들 말 들어봐 - CPU 온도 80도 안 넘는다고"

[20-30초] 🔥 나만의 킥
────────────────────────────────────────
{가격 추적 데이터} + {솔직한 단점}
예) "내가 한 달 추적했는데 지금 140만원이 최저야. 
     무게 2.4kg 무거워, 근데 그 안에 쿨링 다 들어간 거야"

[30-40초] 🎬 타겟 + CTA
────────────────────────────────────────
{REVIEW_AGENT.target} + {최저가 링크 유도}
예) "가성비 겜트북 찾는 사람? 이거야. 
     최저가 링크 걸어뒀어, 가격 더 떨어지면 불러"
```

【품질 검증 체크리스트】
□ 첫 3초 안에 호기심 유발했는가?
□ 스펙이 혜택 언어로 번역되었는가?
□ 실제 리뷰가 최소 1개 인용되었는가?
□ 단점이 솔직하게 언급되었는가?
□ 가격 추적 데이터가 포함되었는가?
□ 과장 표현("대박", "미쳤다") 최소화했는가?
□ 40초 내로 완결되는가?


================================================================================
                    ■ 예시: 레노버 LOQ 15AHP10 대본 ■
================================================================================

【SPEC AGENT 출력】
- RTX 5060 → 최신 게임 풀옵
- 144Hz 디스플레이 → 부드러운 화면
- Hyperchamber 쿨링 → 온도 80도 이하
- 140만원 (6% 할인) → 가성비 최고

【REVIEW AGENT 출력】
- 핵심 인용: "CPU GPU 온도 80도 안 넘어"
- 만족도: 91% 5점
- 구매자: 600명 이상 (한 달)
- 타겟: 게이밍 83%

【STYLE AGENT 출력】
- 후킹: 질문형 시작
- 킥: 가격 추적 데이터 + 솔직한 무게 단점

【BOSS AGENT 최종 대본】
────────────────────────────────────────────────────────────────────────────

가성비 게이밍 노트북 뭐 사야 돼?

쿠팡 리뷰 157개 직접 다 봤어.

게임 풀옵 돌려도 끊김 없고요

실제 산 사람들 말이 
"CPU GPU 온도 80도 안 넘는다"

무게? 솔직히 2.4kg 무거워
근데 그 안에 쿨링 시스템 다 들어간 거야

내가 한 달 가격 추적했는데

【AI에게 대본 요청 시 사용할 프롬프트】

```
당신은 유튜브 쇼츠 대본 작성 멀티에이전트 시스템입니다.

## 입력 데이터
- 제품 스펙: [productinfo.txt 내용]
- 리뷰 분석: [review.txt 내용]  
- 가격 데이터: 현재가 {X}원, 최저가 {Y}원, 구매자 {Z}명

## 에이전트 역할
1. SPEC AGENT: 스펙을 혜택 언어로 번역
2. REVIEW AGENT: 실제 리뷰에서 핵심 인용 추출
3. STYLE AGENT: 차별화 포인트 제안
4. BOSS AGENT: 최종 40초 대본 작성

## 차별화 포인트 (필수 포함)
- 가격 추적 데이터 언급
- 솔직한 단점 1개 이상
- 실제 리뷰 인용 1개 이상
- 구매 조건/가격 기준 명시

## 출력 형식
1. 각 에이전트 분석 결과 (간략히)
2. 최종 대본 (40초 분량)
3. 추천 영상 제목 3개
```
